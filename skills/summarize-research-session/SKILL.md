# Summarize Research Session

## Overview

Produce a concise, structured ~1 page summary from research session notes and/or transcript. Optimized for scanning and cross-session synthesis.

## Inputs

Ask the user to provide:
1. **Session notes and/or transcript** (pasted or as a file)
2. **Session type** — interview, usability test, or other
3. **Hypotheses** (optional) — what they expected going in. If not provided, ask once. If the user says "none," skip the hypothesis sections.

## Output Template

Use this structure for every summary. Keep everything as bullet points — no paragraphs. Omit a section only where noted.

### Participant Background
- 2-3 bullets: role, context, relevant demographics or experience level

### Key Quotes
- 3-5 verbatim quotes from the participant
- Label each with the topic it relates to
- Format: **[Topic]** "Exact quote here"

### Themes
- Patterns, recurring topics, or notable areas of emphasis
- Bullet list

### Usability Issues
- Bullet list of issues observed, each with severity: **High** / **Medium** / **Low**
- *Omit this section entirely if the session was purely interview-based with no task observation*

### Confirmed Hypotheses
- Format: **Hypothesis** — supporting observation
- *Omit if no hypotheses were provided*

### Challenged Hypotheses
- Format: **Hypothesis** — contradicting observation
- *Omit if no hypotheses were provided*

### Additional Observations
- Anything that doesn't fit above: surprises, tangential insights, emotional reactions, contextual notes
- This section is freeform

### Action Items
- Follow-ups, things to flag, next steps

## Guidelines

- Prefer the participant's own words over paraphrasing
- If a quote or observation could fit multiple sections, flag it to the user rather than guessing
- Keep the total summary to roughly 1 page of bullet points
- Do not editorialize or interpret beyond what the data supports
- When severity of usability issues is ambiguous, ask the user rather than assuming
